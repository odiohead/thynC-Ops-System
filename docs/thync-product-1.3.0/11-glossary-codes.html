1. 일반 용어집 (도메인 약어·용어)

thynC 도메인에서 반복 등장하는 제품·임상·기술 용어를 정리합니다. 코드성 값은 이후 섹션(2~11)에서 별도로 다룹니다.

2. 계정 권한 등급 코드

사용자 계정 등급은 user_account.level 정수값으로 표현됩니다. 값이 클수록 상위 권한이며, 콘솔별 접근 범위가 다릅니다.

level코드/명칭접근 범위
1환자 (PATIENT)환자 앱 전용 (콘솔 로그인 대상 아님)
2간호사 (NURSE)서비스 콘솔 — 본인 소속 병동만 조회. 환자관리·모니터링·의료기기 조회
5의사 (DOCTOR)서비스 콘솔 — 전 병동 조회 (user_account 시드에서는 doctor=4로 정의되어, 프론트 표기 5와 차이 존재 — 등급 경계값은 코드 확인 필요)
8매니저/관리자 (MANAGER)서비스 콘솔 — 위 권한 + 관리자 설정 메뉴(기관·계정·병동·알람·장치·게이트웨이·EMR·중앙모니터) 노출
13부관리자 (SUB ADMIN) (추정)관리자 하위 스코프 (DB 정의상 존재, 프론트 미표기)
14관리자 (ADMIN)기관 관리자. 마스터 계정 admin.seers·admin.thync 시드 등급
20시스템관리자 / 사이트매니저 (SYSTEM_ADMIN)THYNC MANAGER(관리자 콘솔) 접속 최소 등급. 미만이면 "시스템 관리자만 접속할 수 있습니다" 차단. 시드 계정 admin.thync10(게이트웨이 등록용)
주의. 프론트엔드 표기(간호사2/의사5/매니저8/관리자14/시스템관리자20)와 DB 시드(user_account.level: patient1/nurse2/doctor4/sysmanager10/subadmin13/admin14/gateway20)의 숫자 매핑이 일부 다릅니다. 권한 판정은 반드시 소스의 등급 상수를 기준으로 확인하세요. 별도 system_manager 컬럼(1:normal, 10:manager)도 존재합니다.

3. 알람 레벨·색상 코드

알람은 레벨(우선순위)과 색상으로 구분되며, 청각 알람(beep)이 동반됩니다. 레벨 값은 bio_signal_alarm_level.alarm_level 기준입니다.

알람 레벨 코드

레벨색상/명칭의미
0None알람 없음 / 기타 알람(ETC_ALARM)
1Red (빨강 #DD1111)최고 우선순위. 응급 부정맥(생명 위협) — 빠른 beep
2Orange (주황) (중간)중간 우선순위 (프론트 알람레벨 상수에서는 "중간=2"로 표기)
3Yellow (노랑 #FF9900)일반 부정맥 또는 생체신호 계산값의 설정 한계 이탈 — 느린 beep
4Fall Detection (낙상)낙상 감지 알람 (시드 FALL_DETECTION 레벨 4). 화면에는 Red로 표시

색상별 의미·사운드

색상대상사운드
Red응급 부정맥: Asystole, VTach(VT), Pause, Extreme Brady, Extreme Tachy, Run PVCs High빠른 beep (base64 내장 오디오, 상시)
낙상 Red낙상 발생 (모니터링 카드·환자 상세 모두 Red)보통 속도 beep (기관 토글 BEEP.FALL_BED)
Yellow일반 부정맥: SVT, Pair PVCs, Bigeminy, Trigeminy, Non-sustain VT, AFIB, Missed beat, Vent Rhythm, High/Low HR, PVCs/min High. 또는 생체신호 한계 이탈느린 beep (기관 토글 BEEP.YELLOW)
Blue기술 알람(장치 관련, DEVICE_ALARM 계열)기관 토글(BEEP.IS_USE.BLUE)에 따라 사용

환자별로 알람 일시정지(Pause) / 알람 확인(Clear) 지원. 부정맥 표시는 blink(깜빡임). 대시보드는 볼륨 조절바 제공.

AFIBYellow(3)심방세동(Atrial Fibrillation)

HIGH_HRYellow(3)고심박(성인 기본 120bpm)

6. 기기(장비) 종류 코드

측정 디바이스는 device_type(정수)으로 분류되며, 콘솔은 시리얼 첫자리로 종류를 자동 판별합니다.

device_type종류대표 제품

제품종류MAC prefix비고

시리얼 첫자리 규칙. 환자등록 시 시리얼번호는 알파벳 1자 + 숫자 6자리 형식으로 검증됩니다(정규식 A/C/P 계열). 콘솔은 이 첫자리와 MAC prefix로 기기 종류를 자동 분류합니다. 첫자리→종류의 정확한 매핑(예: A=ECG, P=SpO2, C=TEMP)은 (추정)이며 소스의 분류 로직 확인이 필요합니다. MAC prefix(제조사 08:D5:C0 OUI + 종류 코드)는 매니저 콘솔 등록 로직에서 확인된 값입니다.

배터리 부족 기준: ECG·TEMP·SpO2 = 2750mV, 반지형 혈압 = 30%, 참케어 = 25%. 공장 마스터(device_factory) 서비스타입: NORMAL/PREMIUM/REPLACEMENT/SAMPLE/DEVELOPMENT.
