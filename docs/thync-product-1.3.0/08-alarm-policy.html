① 알람 체계 개요

thynC 알람은 우선순위에 따라 Red(응급) > 낙상 Red > Yellow(일반) > Blue(기술) 4단계로 구분됩니다. 모니터링 카드는 색상·깜빡임(blink)으로, 청각은 내장(base64) 오디오로 표현하며 대시보드에서 볼륨을 조절합니다.

구분색상우선순위사운드대상레벨코드
Red alarm빨강 #DD1111최상빠른 beep응급 부정맥 (Asystole·VTach·Pause 등)RED = 1
낙상 Red alarm빨강상 (낙상)보통 속도 beep낙상 발생 (카드·상세 모두 Red)Fall = 4
Yellow alarm노랑 #FF9900중느린 beep일반 부정맥 + 생체신호 한계 이탈YELLOW = 3
Blue alarm파랑하 (기술)기관 토글(BEEP.IS_USE.BLUE)장치 관련 기술 알람 (LEAD/배터리 등)Orange = 2 (중간 레벨)

알람 레벨은 DB bio_signal_alarm_level.alarm_level에 정의: 0=None, 1=Red, 2=Orange, 3=Yellow, 4=Fall Detection. 프론트 상수는 RED=1 · (중간)=2 · YELLOW=3으로 매핑됩니다. Yellow·Blue·낙상 사운드는 기관별 BEEP.IS_USE.* 토글로 on/off, 환자별 알람 일시정지(Pause)·알람 확인(Clear)을 지원합니다.

그 밖의 환자 상태 표시: Stand by(검사 등 측정 일시중단) / 등록 장치 없음 / 빈 병상.

② 부정맥 종류별 분류표

분류 근거·주의. Red/Yellow 배정은 프론트 상수(js/custom/constant/monitoring/arrhythmia.js)와 DB 시드(bio_signal_alarm_level: ASYSTOLE·PAUSE·BRADY·TACHY·VT=레벨1, AFIB·HIGH_HR 등=레벨3)를 종합한 것입니다. 항목별 알람 레벨과 on/off는 기관·병동 설정(bio_signal_alarm_level·arrhythmia_alarm)에서 재정의될 수 있으므로, 실제 운영 레벨은 병원 알람 설정을 우선합니다. 알람 카테고리 트리(arrhythmia_category)는 All Alarms / Life-threatening / Ventricular alarms / AFib / Dysrhythmias 5개 대분류로 구성됩니다.

③ 생체신호 한계값 알람 (Limit Alarm)

각 생체신호는 알람 ON/OFF 플래그 + 하한(min)/상한(max) Limit 구조로 관리됩니다. 이탈 시 Yellow 계열 알람이 발생하며, 한계값은 병동 기본값(organization_alarm_setting) → 환자별 개별 설정(measurement_alarm_setting) 순으로 재정의됩니다.

④ EWS(조기경보점수) 개요

EWS(Early Warning Score)는 생체신호를 종합해 환자 악화 위험을 점수화한 값으로, 시계열(ews 테이블)로 저장·트렌드 표시됩니다. HR·RESP·SpO2·TEMP·혈압 등 활력징후 구간별 배점을 합산하는 방식(구체 배점표는 코드에서 확인되지 않음 — 추정)이며, ews_alert_alarm 임계 초과 시 알람을 발생시킵니다.

항목내용
저장ews 시계열(주차 샤드) — value(int 점수), ews_msg(JSON, 산출 근거), ecg_json(구간 ECG)
표시모니터링 카드·대시보드 셀·검토(트렌드) 화면에 EWS 값 표시
알람ews_alert_alarm ON 시 임계 초과 경보 (병동/환자 설정)
조회 APIPOST/API/BioSignal/SelectEwsList · /SelectEwsPage · /SelectEwsLast

⑤ 낙상 알람

낙상은 웨어러블 패치의 자이로·가속도 센서 데이터(gr_ac_json_data)를 기반으로 검출하며, 검출 시 낙상 Red 알람(시·청각)을 발생시킵니다.

항목내용
기능 토글기관 기능환경 FUNCTION_ENVIRONMENT.FALL_BED + 백엔드 bio.signal.fall.event.enabled (병원별: sev=false, dev/premium=true). 환자별 사용 여부는 등록 시 낙상 옵션(measurement_info.fall_use_info)
알람 레벨bio_signal_alarm_level.alarm_level = 4 (Fall Detection). 사운드는 보통 속도 beep(기관 BEEP.IS_USE.FALL_BED 토글)
상태 관리bio_signal_last_fall_event.fall_event_check: 2=대응중 / 1=경보해제 / 0=NONE
대응 처리POST/API/Measurement/UpdateMeasurementFallResponseStatus (낙상 대응 상태 갱신). 실시간 이벤트 101(낙상대응)로 화면 동기화

낙상 원시 파일은 /API/BioSignal/SelectGyroAccelerationRawFile·DownloadGyroAccelerationRawFile로 열람하며, 검출 이벤트는 낙상 카드/환자 상세에 Red로 표시됩니다.

⑥ 알람 운영

일시정지 (Pause)

환자 단위로 알람을 일시 정지합니다. 상태 변경은 실시간 이벤트 50(일시정지)로 다른 클라이언트에 동기화(pause-change)됩니다.

알람 확인/해제 (Clear)

발생 알람을 확인 처리하면 지정 시간(alarm_clear_duration, 기본 5분) 동안 동일 알람을 억제합니다.

⑦ 부정맥 판정 파라미터·설명 (기초 데이터)

부정맥 검출 임계값은 arrhythmia_value_parameter에 성인/소아 기본값을 분리해 시드(REF_ORG 기준 19건)하며, 신규 기관 생성 시 REF_ORG 값을 복사해 초기화합니다. 아래는 시드 기본값입니다.

파라미터 메타는 arrhythmia_value_parameter에 value_data_type(0 INT / 1 BigDecimal), pre_label/post_label(단위 표기), adult_default_value/child_default_value로 저장됩니다. 각 파라미터·부정맥 타입 설명은 다국어(arrhythmia_value_parameter_description: ko 19 + en 19건, arrhythmia_description: KR 17건)로 관리됩니다. 성인/소아 구분은 측정 세션의 age_group_type(1 Adult / 2 Pediatric·Neonatal)을 따릅니다.

임상 주의. 위 값은 REF_ORG 시드 기본값이며, 병원·병동·환자별로 재정의됩니다. 실제 적용 임계값은 반드시 해당 측정의 arrhythmia_alarm/arrhythmia_value_parameter 조회로 확인하세요.

⑧ 장치 상태·기술 알람

웨어러블 장치의 연결·리드·배터리 상태는 Blue(기술) 알람으로 통지됩니다. 상태는 device_last_status_info에 측정코드+타입당 1행으로 유지되고, 실시간 이벤트 80(장치 알람)으로 화면에 반영됩니다.

리드 부착LEAD ON2 (lead on)
