1. 설정 체계 개요

설정은 공통 파일 + 프로파일별 파일 2계층으로 구성됩니다.

파일범위주요 내용
application.properties공통(모든 프로파일)빌드 메타(ext.*), spring.profiles.active, server.port=80, JPA 공통(ddl-auto=none, MySQL5Dialect, open-in-view=true), SMTP(Gmail 587), multipart 100MB, 공통 로깅 레벨, SkyLabs 기본
application-thync-api-<프로파일>.properties프로파일별(전체 설정)인프라·DB·Redis·Kafka·DICOM·EMR·HL7·임상 정책 등 프로파일 단위 전체 오버라이드. 총 17종

버전 메타(ext.*)가 빌드 버전의 원천입니다(MANIFEST.MF에는 버전 없음). 활성 프로파일은 spring.profiles.active 한 줄로 선택합니다.

# application.properties
spring.profiles.active=thync-api-onpremise
ext.buildVersion=01.01.431   # (표기 예 — ext.* 키가 버전 원천)

2. 프로파일 구조

총 17종의 프로파일이 4개 계열로 나뉩니다.

계열프로파일인프라특징
클라우드dev, test, premiuminfrastructure=cloudAWS Aurora RDS / S3 저장 / ElastiCache(test), read 풀 활성, Kafka 토픽 .dev, 스케줄러 test-config
온프레 기본onpremiseinfrastructure=onPremise표준 템플릿, 전부 localhost. 본 문서 3절 기본값의 기준
온프레 병원별onpremise-{sev, cnuhh, bjh, bs, believesewoong, gg, gh, gnasan, hns, icb, mch, ezcaretech}onPremise기본 대비 diff로 병원별 커스텀(4절). 총 12종
특수local-ezcare-release로컬이지케어텍 연동 로컬 릴리즈 테스트
일부 병원 파일(sev 등)은 CRLF 개행으로 Windows에서 편집된 흔적이 있습니다. 편집 환경 차이로 인한 개행 혼재는 diff 노이즈를 유발하니 편집 도구 개행 설정에 유의하세요.

4. 병원별 커스텀 프로파일 비교

기준: onpremise 기본 대비 diff. 병원명은 이니셜 기반 추정이며 근거 강도를 표기합니다(★근거 명확 / ☆추정).

프로파일병원 (추정)핵심 커스텀
sev세브란스병원 ☆emr.organization.code=THYNC, cardio.send.type=false·ssl.disabled=false, 호흡/낙상 이벤트 명시 false. 그 외 기본과 동일(최소 커스텀)
cnuhh화순전남대병원 ☆HR 소스 spo2, 체온 AI 보정(=1), read 풀 활성(로컬), Hikari 타임아웃 구세대 값, net.seers DEBUG. 구형 포맷(212줄 — report/bprawfile/보존기간 키 부재 → 코드 기본값)
gh미상(G병원)cnuhh 계열: HR 소스 spo2, read 풀, DEBUG 로깅. 구형 포맷(222줄)

병원 커스텀 5개 축

① EMR 전송on/off·기관코드
② HL7 인터페이스TCP(MLLP)/SFTP
③ DICOM PACSAE/host/port
④ Oracle HIS직결 조회/전송
⑤ 임상 정책HR소스·트렌드·알람

DB·Redis·디렉토리 등 인프라 설정은 전 병원 동일(localhost 표준 구성)하며, 병원별 차이는 위 5개 축에 집중됩니다.

5. 설정 위생 유의사항

비밀값 취급 원칙. 본 문서 전반의 비밀값은 마스킹(**********) 처리되어 있습니다. 실제 프로퍼티 파일에는 자격증명이 평문으로 존재할 수 있으나, 문서·위키·이슈 등 외부 공유물에는 실제 값을 절대 기재하지 마십시오.

운영 안정성과 보안을 위한 개선 권고 사항입니다(현행 동작에는 문제 없으나 정비 대상).

항목현황개선 권고
평문 자격증명 내장DB·Redis·JMX·Gmail·AWS·이지케어텍 시크릿이 프로퍼티에 평문 존재환경변수·시크릿 관리(예: Jasypt/외부 시크릿 스토어)로 분리 (추정 방안)
미사용 AWS 키 잔존온프레(LOCAL)에도 AWS 액세스키·S3 버킷 값 잔존온프레 프로파일에서 미사용 클라우드 자격증명 제거
설정 오타 키project.sever.type, ...listener.sever.autostartup(sever), project.log.base.folder(로고 용도), tempaaaa/, JMX port 9090AAAAAAA오타 정정 시 하위호환 확인 후 일괄 정비(현재 코드가 오타 키에 의존)
값 복사 흔적bjh DICOM org=MCH 등 타 병원 값 복사 잔재병원별 값 재검증
개행 혼재일부 병원 파일 CRLF(Windows 편집)편집 도구 개행 통일(LF)로 diff 노이즈 제거
JSON 로그 미연결JSON_FILE appender 정의만 있고 root 미연결구조화 로그 필요 시 root 연결, 불필요 시 정의 제거
