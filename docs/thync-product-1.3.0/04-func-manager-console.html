1. 개요

1.1 제품 정의

thynC Manager Console은 SEERS사의 병원 입원환자 무선 웨어러블 생체신호 모니터링 솔루션(thynC / mobiCARE Console)의 본사 시스템 관리자 전용 관리 콘솔이다. 개별 병원(기관)에서 사용하는 임상 콘솔과 달리, 이 콘솔은 전 병원에 공통으로 적용되는 마스터 데이터·인프라(게이트웨이·의료기기·기관·계정·펌웨어·SW버전·공지·통계)를 SEERS 본사 담당자가 일괄 관리하는 용도이다. 사이트 타이틀은 "THYNC MANAGER"이며, 저작권 표기는 Copyright 2023 Seers Technology이다.

1.2 대상 사용자 및 접속 차단 로직

대상 사용자는 SEERS 본사의 시스템 관리자 전용이다. 로그인 성공 후 checkPostAuth()가 계정 등급을 검사하여, 계정 level이 20(SYSTEM_ADMIN) 미만이면 "시스템 관리자만 접속할 수 있습니다!" 메시지를 표시하고 접속을 차단한다. 즉 병원 측 간호사·의사·매니저·관리자 계정으로는 로그인 자체는 통과해도 콘솔 진입이 봉쇄된다.

1.3 기술 개요

  • 빌드 과정이 없는 Vanilla JS SPA (자체 프레임워크 "seers frontframework for vanilla"). ES 동적 import() + 자체 파일 로더(preFileLoading)로 controller / template / module을 지연 로딩한다.
  • 라우팅: URL 경로 /{controller}/{action} 구조 (선택적 prefix GBL.MANAGER_PREFIX = "/manager"). js/common/loader.js가 경로를 파싱하여 js/controller/{name}.js + template/thync-manager/content/{name}.js를 로드한다. 기본 컨트롤러는 gateway, 기본 액션은 index이다.
  • 공통 CRUD 모듈: js/module/crud/ — 화면별 설정 파일(js/module/crud/init/thync-manager/*.js)의 선언(검색폼 컬럼·테이블 컬럼·버튼·API 매핑·콜백)만으로 목록/검색/페이징/등록/수정/삭제 화면을 자동 생성한다. 페이징은 10건 단위.
  • 주요 라이브러리: jQuery 3.5.1(+cookie·UI 1.12.1·timepicker), Bootstrap 3.3.2, Handlebars 4.7.7(템플릿), Chart.js 4.4.3(통계 차트), ExcelJS 4.3.0(엑셀 import/export), asmCrypto 0.22.0(SHA-256 파일 체크섬), SockJS 1.5.0 + STOMP 1.7.1(스트림 서버용 socket 모듈), slimscroll·jquery.toast·date.js(ko-KR).
  • 공통 모듈: layout(헤더·좌측메뉴), login, paging, popup, modal(custom-alert), toast-message, version, window-worker.

1.4 API 베이스 URL

모든 API는 GBL.API.SET_BASE_URL()이 환경별로 결정하며, 베이스 경로는 .../mobiCAREConsole/API이다.

환경베이스 URL
운영https://www.api.seersthync.com/mobiCAREConsole/API
개발https://www.dev-api.seersthync.com/mobiCAREConsole/API
온프레미스 (GBL.ON_PREMISE=true, 기본값)http://{host}:8080/mobiCAREConsole/API 또는 https://{host}:8443/... (.mex-us.kr 도메인이면 192.168.0.1 고정)
  • 전 API POST + JSON (파일 업로드는 FormData).
  • 요청 헤더: SX-Auth-Token(로그인 토큰), SX-Client-IP.
  • 공통 파라미터 자동 첨부: requester(userCode), organizationCode, deviceKind=3, timezone="Asia/Seoul", gmtCode, requestDateTime, systemTime, countryCode/Name.

1.5 인증 흐름

1. 로그인/login → /Account/LoginHIS
2. 등급 검사checkPostAuth() level ≥ 20
3. 토큰 저장userAccount·accessToken 쿠키
4. 랜딩/gateway/index
  • 로그인 성공 시 userAccount(계정정보 JSON)와 accessToken을 쿠키에 저장하고, 이후 매 요청에서 SX-Auth-Token 헤더로 사용한다. 아이디 저장(30일) / 자동 로그인(365일) 쿠키 옵션 제공.
  • 세션 갱신 interval은 2시간(GBL.SESSION_RENEW). 1초 주기로 토큰 존재를 체크하여 없으면 로그인 화면으로 이동.
  • 로그인 실패 상세 메시지: 계정 없음 / 비밀번호 오류 / 기관 사용 종료(Organization is locked) / 계정 잠금 / 시스템 관리자 아님(Invalid user level) / 허용되지 않은 IP(Access denied).
  • 삼성서울병원 전용 빌드에서는 로그인 API가 /Account/LoginHISforManager로 대체된다(주석 형태로 존재).

4. 화면별 기능 정의 (18종)

4.1 로그인

js/module/login/*, template/thync-manager/content/login.js

  • 목적: 시스템 관리자 인증
  • 주요 기능: ID/PW 입력, 아이디 저장, (옵션)자동 로그인, Enter 키 로그인, 하단 버전/저작권 표시(version 모듈)
  • 표시 데이터: 로고(텍스트), 실패 사유별 메시지(계정없음/비번오류/기관잠금/계정잠금/권한부족/IP차단), 비밀번호 변경주기 알림(옵션, 90일)
  • 사용 API: POST/Account/LoginHIS (삼성전용: /Account/LoginHISforManager)

4.2 게이트웨이 목록

/gateway/index, js/module/crud/init/thync-manager/gateway.js

  • 목적: 병원(기관)별 게이트웨이(무선 수신 장치) 현황 관리
  • 주요 기능:
    • 검색: 기관 셀렉트박스 + 검색어(기관 이름)
    • 목록 자동 새로고침(Refresh 타이머) + 수동 "새로고침" 버튼, 마지막 갱신시각 표시
    • 등록 / 대량 등록(/gateway/dynamic?sub=add-bulk) / 행별 수정·삭제 / 체크박스 선택 삭제 / 선택 이동(체크한 게이트웨이들을 다른 기관으로 일괄 이관 — 대상 기관 선택 모달)
  • 표시 데이터(테이블): 체크박스, 번호, 기관, 연결상태(연결/미연결), serial Number, macAddress, 마지막 연결시간(connStatusUpdateTimeByStream), 펌웨어버전, 설치장소(층), 비고, 관리(수정/삭제)
  • 사용 API: /Manager/SelectGatewayInfoPage(목록), /Manager/SelectGatewayInfo(List), /Manager/DeleteGatewayInfo, /Manager/DeleteGatewayInfoList(선택삭제), /Manager/UpdateAllGWOrganizationCodeList(선택이동), 기관목록 /Manager/SelectOrganizationList

4.3 게이트웨이 등록/수정

/gateway/add, /gateway/update

  • 입력 필드: 기관(필수, 셀렉트), 시리얼넘버(필수, 7자리 — macAddress는 시리얼번호에서 자동 계산 08:D5:C0:2 prefix), Firmware Version(필수), 설치 장소(층, 8자 이내), 설치 장소 X/Y 좌표(숫자 검증), 비고
  • 검증: 시리얼 중복 검사(이미 타 기관에 등록 시 기관명 안내), X/Y 숫자 검사
  • 사용 API: /Manager/InsertGatewayInfo, /Manager/UpdateGatewayInfo

4.4 게이트웨이 대량 등록

/gateway/dynamic?sub=add-bulk

  • 주요 기능: 시리얼번호 개별 추가(칩 UI) 또는 엑셀 파일 업로드(첫 시트 A열 시리얼 목록, ExcelJS 파싱, 2007+ xlsx만), 중복/형식 검사 후 일괄 등록
  • 사용 API: /Manager/InsertGatewayListInfo

4.5 게이트웨이 펌웨어

/gateway-fw/index, /gateway-fw/add, js/module/crud/init/thync-manager/gateway-fw.js

  • 목적: 기관별 게이트웨이 펌웨어 파일 배포 관리
  • 주요 기능: 목록 조회, 등록, 행별 삭제. 등록 시 펌웨어 파일 업로드(FormData) + SHA-256 체크섬 클라이언트 계산(asmCrypto) 후 함께 전송
  • 표시 데이터: 기관, 버전(fwVersion), 크기(fileSize), 레벨(Normal/High/Instant — 업데이트 우선순위), 등록시간, 비고, 관리(삭제)
  • 등록 폼: 기관(셀렉트), 버전, 레벨(라디오 Normal/High/Instant), 파일(파일찾기), 비고
  • 사용 API: /Manager/SelectGatewayFwInfoPage, /Manager/SelectGatewayFwInfo(List), /Manager/InsertGatewayFwInfo, /Manager/DeleteGatewayFwInfo (UPDATE 슬롯은 /Manager/UpdateGatewayInfo에 매핑되어 있으나 실사용 없음)

4.6 계정 관리

/account/index, /account/add, /account/update, js/module/crud/init/thync-manager/account.js

  • 목적: 전 기관 사용자 계정(간호사/의사/매니저/관리자 등) 관리
  • 주요 기능:
    • 검색: 기관 셀렉트 + 검색어. 목록 페이징
    • 등록: 기관, 계정명(ID), 계정명(닉네임), 이메일, 소속병동(기관 선택 시 병동 목록 동적 로드), 계정권한(환자/간호사/의사/매니저/관리자/사이트매니저 라디오), 비밀번호/비밀번호 확인 (규칙: 8자 이상 영문+숫자, 개인정보강화 시 특수문자 포함)
    • 수정: 기본정보 수정 + 비밀번호 변경(현재 비밀번호 검증 /Account/CheckPassword → 새 비밀번호/재입력, 현재와 동일 금지)
    • 삭제/선택 삭제: 확인 모달, 개인정보강화 시 감사 메모(auditMemo) 입력 UI + 동일 기관 건만 일괄 처리
    • 개인정보강화 시 이메일 마스킹 표시
  • 표시 데이터: 체크박스, Id, 닉네임, email, 기관, 권한, 관리(수정/삭제)
  • 사용 API: /Account/SelectAccountSimplePage, /Account/SelectAccountSimple, /Account/SelectAccountSimpleList, /Account/SelectAllAccountSimplePage, /Account/SelectAllAccountSimpleList, /Account/CreateAccount, /Account/UpdateAccount, /Account/UpdatePassword, /Account/CheckPassword, /Account/DeleteAccount, /Account/DeleteAllAccountList, /Manager/SelectSystemManagerList, 병동 /Manager/SelectWard, 기관 /Manager/SelectOrganizationList

4.7 기관 관리

/organ/index, /organ/add, /organ/update, js/module/crud/init/thync-manager/organ.js

5. 공통 기능

5.1 CRUD 공통 모듈 (선언형 화면 생성)

거의 모든 목록/등록/수정 화면은 js/module/crud/의 공통 엔진으로 생성된다. 화면별 설정 파일(js/module/crud/init/thync-manager/*.js)에 선언(검색폼 컬럼, 테이블 컬럼, 버튼, API 매핑 CREATE/READ/UPDATE/DELETE 슬롯, 콜백)만 기술하면 목록·검색·페이징(10건 단위)·등록·수정·삭제 화면이 자동으로 조립된다. SW버전·스카이랩스 기관 등 특수 화면은 별도 커스텀 컨트롤러(js/controller/*.js)로 구현된다.

5.2 엑셀 import / export

ExcelJS 4.3.0을 사용한다. 게이트웨이·의료기기 대량 등록은 엑셀 파일(2007+ xlsx만) 첫 시트 A열의 시리얼 목록을 파싱하여 칩 UI로 취합 후 일괄 등록한다. 형식/중복 검사를 통과한 항목만 전송된다.

5.3 SHA-256 파일 체크섬 업로드

펌웨어·SW버전 파일 업로드 시, 클라이언트에서 asmCrypto 0.22.0으로 SHA-256 체크섬을 계산하여 파일(FormData)과 함께 전송한다. 서버 측 무결성 검증 용도(추정)이다.

5.4 시리얼 → macAddress 자동 계산 규칙

게이트웨이 및 의료기기는 입력한 시리얼번호로부터 macAddress를 자동 계산한다. 확인된 prefix 체계는 공통적으로 08:D5:C0:x 형태이며, 장치 종류에 따라 마지막 자리(x)가 구분된다.

대상prefix
게이트웨이08:D5:C0:2
의료기기 Tag08:D5:C0:1
의료기기 SpO208:D5:C0:4
의료기기 ECG08:D5:C0:5
의료기기 TEMP08:D5:C0:6
의료기기 BP08:D5:C0:7

Ring BP(SkyLabs)는 자동계산 대상이 아니며 SkyLabs API가 제공한 값을 사용한다. 위 표는 소스에서 확인된 범위에 한정한다.

5.5 세션 · 권한

  • 로그인 토큰(accessToken)은 쿠키에 저장되고 매 요청 SX-Auth-Token 헤더로 전송된다. 세션 갱신 interval 2시간, 1초 주기 토큰 존재 체크.
  • 응답 코드 0x00F1(SESSION_CLOSED) 수신 시 즉시 로그인 화면으로 이동한다.
  • 계정 등급 20(SYSTEM_ADMIN) 미만은 checkPostAuth()에서 콘솔 진입이 차단된다(1.2 참조).
