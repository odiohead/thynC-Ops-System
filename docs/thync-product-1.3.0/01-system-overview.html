1. 제품 개요 · 목적

thynC(정식 명칭 thynC IP2140, SEERS社 mobiCARE Console)는 병원 입원 환자를 대상으로 한 무선 웨어러블 생체신호 실시간 모니터링 솔루션입니다. 식약처 품목허가 제인 23-5092호(환자감시장치 소프트웨어)로 허가된 의료기기 소프트웨어입니다.

기존 병상 감시 장치(bedside monitor)는 환자를 침상과 케이블에 묶어 두어 거동을 제약하고, 일반 병동(General Ward)에는 설치 비용 문제로 보급되기 어려웠습니다. thynC는 이 공백을 경량 무선 웨어러블 + 게이트웨이 무선 수신 + 중앙 콘솔 통합 감시 구조로 해결합니다. 환자는 케이블 없이 병동을 자유롭게 이동하고, 의료진은 여러 병동의 다수 환자를 하나의 중앙 콘솔에서 동시에 감시합니다.

동작 흐름 (개념)

환자에게 심전계(ECG)·펄스옥시미터(SpO2)·체온계·혈압계(참케어형/반지형) 등 웨어러블 센서를 부착하면, 센서가 측정한 생체신호를 병동에 설치된 게이트웨이가 무선으로 수신합니다. 게이트웨이는 이를 원내 API 서버로 전달하고, 서버는 신호를 저장하고 AI 부정맥 분석과 알람 임계값 판정을 수행합니다. 판정 결과와 실시간 파형은 WebSocket으로 간호사·의사의 서비스 콘솔에 즉시 전송되어 중앙 감시 화면에 표시됩니다.

게이트웨이무선 수신 · 서버 전달
API 서버저장 · AI 부정맥 분석 · 알람 판정
서비스 콘솔간호사 · 의사 실시간 감시

제품이 해결하는 핵심 가치는 다음과 같습니다.

과제thynC의 해결 방식
일반 병동의 상시 감시 부재무선 웨어러블로 케이블·설치 부담 없이 병동 단위 상시 모니터링
이상 징후의 지연 인지AI 부정맥 분석 + 임계값 알람으로 실시간 위험 감지·통보
환자 거동 제약무선 센서 + 게이트웨이 수신으로 이동 중에도 신호 수집
분산된 감시 지점중앙 콘솔(카드뷰·대시보드)로 다병동·다환자 통합 감시
병원 시스템과의 단절EMR · HIS · HL7 · DICOM(PACS) 연동으로 원내 워크플로 통합

2. 이해관계자 · 사용자 역할

thynC는 환자·의료진·병원 관리자·제조사(SEERS)를 아우르는 다중 이해관계자 시스템입니다. 서비스 콘솔의 접근 권한은 계정 등급(level/grade)으로 통제되며, 상위 등급이 하위 등급의 기능을 포함합니다. 콘솔 UI 등급값과 DB의 user_account.level 값이 계층상 대응합니다.

6. 배포 형상 (온프레미스 · 클라우드)

thynC는 병원별 온프레미스(원내 서버) 배포가 주력이며, 일부 사이트는 클라우드(AWS)로 운영됩니다. 배포 프로파일은 WAR 한 개에 전부 내장되고, application.properties의 spring.profiles.active 한 줄로 활성 프로파일이 선택됩니다(병원별 WAR = active 프로파일만 다름).

병원별 커스텀은 5개 축으로 나타납니다 — ① EMR 전송 on/off·기관코드, ② HL7 TCP/SFTP 인터페이스, ③ DICOM PACS 전송 대상(AE/host/port), ④ Oracle HIS 직결, ⑤ HR 소스·트렌드·알람 등 임상 정책. DB·Redis·디렉토리 등 인프라 설정은 전 병원 공통(localhost 표준)입니다. 프로파일별 상세는 환경설정 정의서를 참조하세요.

프론트엔드는 웹서버의 ~/html/<DIRECTORY>/ Git 저장소를 git reset --hard로 강제 동기화하여 배포합니다(빌드 없음). 백엔드 반영은 DEV → git → PROD git pull 절차를 따릅니다.

7. 데이터 규모 · 특성

각 측정 세션의 샤드 위치는 measurement_info의 shard(연도)·part(0~100) 컬럼이 라우팅 키로 지정합니다. 모든 시계열 샤드는 measurement_info.measurement_code를 FK(ON DELETE CASCADE)로 참조하므로, 측정 세션 삭제 시 관련 시계열이 일괄 삭제됩니다. 실시간 화면 응답을 위해 최신값 캐시 테이블 10종을 별도 유지합니다.

데이터 보존. 기본 보존 기간은 90일(data.retention.days=90, global_setting.backup_day)이며, old.data.delete.type=1일 때 보존 기간이 지난 DB 행과 파일이 삭제됩니다. ECG 원본 파일은 온프레 LOCAL 디스크, 클라우드 S3에 저장됩니다.
